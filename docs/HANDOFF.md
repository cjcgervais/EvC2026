## вЪ†пЄП THE ONE DECISION ALREADY MADE вАФ read before touching the client

The server (`GameServer`) deliberately sets `player.CharacterAutoLoads = false` and represents each player's bird(s) as **standalone Models in `Workspace.Birds`** вАФ NOT as a Roblox Character with a `HumanoidRootPart`.

**This is the correct architecture and is now canonical.** Do not revert it. Reason: a Crow player controls **4 separate bird bodies** with possession-swapping вАФ a single Roblox Character cannot represent that. The eagle is just the 1-body case of the same model.

**This was integration task #1 and is now DONE in code** (see the Status section above). The implemented contract вАФ the canonical source of truth both sides now honor вАФ is:

### The bird-discovery attribute contract (IMPLEMENTED)
On every bird Model in `Workspace.Birds`, set *before parenting* by `GameServer.spawnBird`:
- `OwnerUserId` (number) вАФ `player.UserId` of the controlling player (`0` if none).
- `Team` (string) вАФ `"Eagles"` / `"Crows"`.
- `Possessed` (bool) вАФ `true` on the single bird the player actively flies; updated by `onSwapCrow` and `autoRepossess`.
On the `Body` part (the `PrimaryPart`): `Health` (number) and `MaxHealth` (number) attributes.
Network ownership of the possessed bird's `Body` goes to the player via `SetNetworkOwner`; AI crows are `SetNetworkOwner(nil)` (server-owned).

The client (`BirdController`) drives the Model whose `OwnerUserId == player.UserId and Possessed == true`, re-acquiring on folder `ChildAdded`/`ChildRemoved` and on any owned Model's `Possessed` attribute change. **Do not break this contract вАФ grep `OwnerUserId` / `Possessed` before touching spawning or possession.**
