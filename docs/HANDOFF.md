## Status (2026-06-28) ‚Äî session 6: FlightPhysics **v2 kernel** rewrite (code-complete, build-green, UNPLAYTESTED)

Addresses the persistent "not enough lift, constantly stalls" complaint at its **root** (the prior `Cl‚Üí2.4` / `flapThrust‚Üí520` / spawn-speed / soft-stall-floor edits were all symptom patches). A fresh deep-research pass on **game flight-model implementation** (Vazgriz, brihernandez/SimpleWings, gasgiant Aircraft-Physics, Aerofly TMD; archived in `docs/RESEARCH.md` ¬ß"v2") confirmed the diagnosis independently and supplied the numbers. The kernel is a **drop-in rewrite** ‚Äî same API + same readable fields, so `BirdController`, `GameServer` AI loop, `Boids`, and `BirdCollision` are untouched. **Git checkpoint committed before the rewrite** (`git log` ‚Üí "Checkpoint before FlightPhysics v2 kernel rewrite") ‚Äî revert `FlightPhysics.luau` + `GameConfig.luau` to roll back.

**Root cause (two flaws fighting each other):** (1) lift was `Cl¬∑q¬∑sin(AoA)` ‚Üí **zero lift at zero AoA**, forcing the bird to fly permanently nose-high a few degrees from stall just to hold altitude; (2) the per-frame **path-tracker rotated velocity toward the nose**, i.e. drove AoA‚Üí0 ‚Üí the no-lift condition. Any speed dip dropped authority, weakening lift+control+tracking together ‚Üí self-reinforcing stall spiral.

**What landed (`FlightPhysics.luau` `:Update`, `GameConfig.luau`):**
1. **Real lift curve with baseline lift** ‚Äî `CL(Œ±)=cl0+liftSlopePerDeg¬∑Œ±`, capped at `clMax` at the stall angle, then a forgiving lerp to `postStallCl` over `stallPadDeg` (a recoverable mush, never 0). `cl0=0.30` is THE fix: the bird makes lift at zero AoA, so **level cruise is the hands-off trim** (verified calibration: Eagle trims ~0.47¬∞ AoA, stall ~65 < spawn 117 < cruise 130; Crow ~0.57¬∞ AoA, stall ~45 < spawn 86 < cruise 95).
2. **Drag polar `CD = CD0 + k¬∑CL¬≤`** (replaces `sin¬≤`) ‚Äî induced drag ‚àù lift¬≤ ‚àù g-load¬≤, so hard turns/climbs bleed energy ‚Üí Energy-Maneuverability emerges. Drag is mass-normalized (heavier eagle retains energy); **lift is mass-independent** (`liftAccel = CL¬∑AIR_DENSITY¬∑v¬≤ / wingLoading`) so `wingLoading` is the stall-speed/turn lever.
3. **Static aerodynamic stability (weathervane) REPLACES path-tracking** ‚Äî orientation rotates toward the airflow at `STABILITY_RATE¬∑stabilityRate¬∑authority`; AoA self-limits (anti-stall) **without** zeroing lift, and player pitch overrides it. Past stall, `recoverNoseDownRate` adds nose-down-toward-airflow so "do nothing" recovers. **Coordinated turn is now bank-gated only** (`sustainedTurnRate`, eagle-wide/crow-tight) ‚Äî never acts in the pitch plane, so it cannot recreate the old bug. `rotateLookToward` preserves position (rotates basis only).
4. **Free-look hardened (it's LAW):** vertical clamp widened ¬±1.3‚Üí**¬±1.54 rad (~88¬∞)** so you can look nearly straight up/down (yaw was already unbounded ‚Üí full 360¬∞); **free-look now survives a crow-swap** (`acquire()` re-arms to whether Space is still held instead of force-off ‚Äî fixes queue #7 bug); added an in-code invariant note that free-look must never write `inputState`.

**New `GameConfig` keys (contract):** per-profile `cl0`, `liftSlopePerDeg`, `clMax`, `postStallCl`, `stallPadDeg`, `stabilityRate`, `recoverNoseDownRate`; `Flight.STABILITY_RATE`; recalibrated `Flight.AIR_DENSITY = 0.033`. **Removed:** per-profile `liftCoefficient` and the dead `stallRecoverRate`; **replaced** `Flight.PATH_TRACK_RATE` with `Flight.STABILITY_RATE`. (Dead levers `AEROBATIC_MIN_SPEED`/`climbCeilingBonus` still present ‚Äî wire or delete in the tuning pass.) All numbers **PLAYTEST-PROVISIONAL.**

**MUST validate in Studio (queue #1 checklist applies, plus these):** (a) Eagle **holds altitude on neutral input / no flap** and only stalls when you yank the nose at low speed, then **recovers by easing off** (mush, not spiral); (b) nose-down accelerates, climb bleeds speed, dive reaches `diveSpeedCap`, banking turns (eagle wide / crow tight); (c) **AI crows hold altitude** (same engine ‚Äî better lift should fix their sink/crash too); (d) **free-look: hold Space ‚Üí look fully up/down/behind/360¬∞ while flying straight; swap crows 1‚Äì4 while holding Space ‚Üí look persists**; release ‚Üí snap back to chase; (e) no spawn/swap teleport (the `rotateLookToward` position fix), anti-cheat doesn't false-positive. **The recalibration theorem (why it was safe):** drop `GRAVITY` and `AIR_DENSITY` by the *same* factor and the flight envelope is invariant ‚Äî in `v_stall = ‚àö(g¬∑WL/(clMax¬∑œÅ))` the factor cancels. **Verified numerically: Eagle stall = 61.0, Crow stall = 45.2 at GRAVITY_G ‚àà {1,2,3} ‚Äî identical**, so `stall < spawn(117/85) < cruise(130/95)` is structurally guaranteed at any gravity. ## ‚öÝÔ∏è THE ONE DECISION ALREADY MADE ‚Äî read before touching the client

The server (`GameServer`) deliberately sets `player.CharacterAutoLoads = false` and represents each player's bird(s) as **standalone Models in `Workspace.Birds`** ‚Äî NOT as a Roblox Character with a `HumanoidRootPart`.

**This is the correct architecture and is now canonical.** Do not revert it. Reason: a Crow player controls **4 separate bird bodies** with possession-swapping ‚Äî a single Roblox Character cannot represent that. The eagle is just the 1-body case of the same model.

**This was integration task #1 and is now DONE in code** (see the Status section above). The implemented contract ‚Äî the canonical source of truth both sides now honor ‚Äî is:

### The bird-discovery attribute contract (IMPLEMENTED)
On every bird Model in `Workspace.Birds`, set *before parenting* by `GameServer.spawnBird`:
- `OwnerUserId` (number) ‚Äî `player.UserId` of the controlling player (`0` if none).
