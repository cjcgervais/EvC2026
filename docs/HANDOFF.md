## Status (2026-06-28) вҖ” session 6: FlightPhysics **v2 kernel** rewrite (code-complete, build-green, UNPLAYTESTED)

Addresses the persistent "not enough lift, constantly stalls" complaint at its **root** (the prior `ClвҶ’2.4` / `flapThrustвҶ’520` / spawn-speed / soft-stall-floor edits were all symptom patches). A fresh deep-research pass on **game flight-model implementation** (Vazgriz, brihernandez/SimpleWings, gasgiant Aircraft-Physics, Aerofly TMD; archived in `docs/RESEARCH.md` В§"v2") confirmed the diagnosis independently and supplied the numbers. The kernel is a **drop-in rewrite** вҖ” same API + same readable fields, so `BirdController`, `GameServer` AI loop, `Boids`, and `BirdCollision` are untouched. **Git checkpoint committed before the rewrite** (`git log` вҶ’ "Checkpoint before FlightPhysics v2 kernel rewrite") вҖ” revert `FlightPhysics.luau` + `GameConfig.luau` to roll back.

**Root cause (two flaws fighting each other):** (1) lift was `ClВ·qВ·sin(AoA)` вҶ’ **zero lift at zero AoA**, forcing the bird to fly permanently nose-high a few degrees from stall just to hold altitude; (2) the per-frame **path-tracker rotated velocity toward the nose**, i.e. drove AoAвҶ’0 вҶ’ the no-lift condition. Any speed dip dropped authority, weakening lift+control+tracking together вҶ’ self-reinforcing stall spiral.

**What landed (`FlightPhysics.luau` `:Update`, `GameConfig.luau`):**
1. **Real lift curve with baseline lift** вҖ” `CL(Оұ)=cl0+liftSlopePerDegВ·Оұ`, capped at `clMax` at the stall angle, then a forgiving lerp to `postStallCl` over `stallPadDeg` (a recoverable mush, never 0). `cl0=0.30` is THE fix: the bird makes lift at zero AoA, so **level cruise is the hands-off trim** (verified calibration: Eagle trims ~0.47В° AoA, stall ~65 < spawn 117 < cruise 130; Crow ~0.57В° AoA, stall ~45 < spawn 86 < cruise 95).
2. **Drag polar `CD = CD0 + kВ·CLВІ`** (replaces `sinВІ`) вҖ” induced drag вҲқ liftВІ вҲқ g-loadВІ, so hard turns/climbs bleed energy вҶ’ Energy-Maneuverability emerges. Drag is mass-normalized (heavier eagle retains energy); **lift is mass-independent** (`liftAccel = CLВ·AIR_DENSITYВ·vВІ / wingLoading`) so `wingLoading` is the stall-speed/turn lever.
3. **Static aerodynamic stability (weathervane) REPLACES path-tracking** вҖ” orientation rotates toward the airflow at `STABILITY_RATEВ·stabilityRateВ·authority`; AoA self-limits (anti-stall) **without** zeroing lift, and player pitch overrides it. Past stall, `recoverNoseDownRate` adds nose-down-toward-airflow so "do nothing" recovers. **Coordinated turn is now bank-gated only** (`sustainedTurnRate`, eagle-wide/crow-tight) вҖ” never acts in the pitch plane, so it cannot recreate the old bug. `rotateLookToward` preserves position (rotates basis only).
4. **Free-look hardened (it's LAW):** vertical clamp widened Вұ1.3вҶ’**Вұ1.54 rad (~88В°)** so you can look nearly straight up/down (yaw was already unbounded вҶ’ full 360В°); **free-look now survives a crow-swap** (`acquire()` re-arms to whether Space is still held instead of force-off вҖ” fixes queue #7 bug); added an in-code invariant note that free-look must never write `inputState`.

**New `GameConfig` keys (contract):** per-profile `cl0`, `liftSlopePerDeg`, `clMax`, `postStallCl`, `stallPadDeg`, `stabilityRate`, `recoverNoseDownRate`; `Flight.STABILITY_RATE`; recalibrated `Flight.AIR_DENSITY = 0.033`. **Removed:** per-profile `liftCoefficient` and the dead `stallRecoverRate`; **replaced** `Flight.PATH_TRACK_RATE` with `Flight.STABILITY_RATE`. (Dead levers `AEROBATIC_MIN_SPEED`/`climbCeilingBonus` still present вҖ” wire or delete in the tuning pass.) All numbers **PLAYTEST-PROVISIONAL.**

**MUST validate in Studio (queue #1 checklist applies, plus these):** (a) Eagle **holds altitude on neutral input / no flap** and only stalls when you yank the nose at low speed, then **recovers by easing off** (mush, not spiral); (b) nose-down accelerates, climb bleeds speed, dive reaches `diveSpeedCap`, banking turns (eagle wide / crow tight); (c) **AI crows hold altitude** (same engine вҖ” better lift should fix their sink/crash too); (d) **free-look: hold Space вҶ’ look fully up/down/behind/360В° while flying straight; swap crows 1вҖ“4 while holding Space вҶ’ look persists**; release вҶ’ snap back to chase; (e) no spawn/swap teleport (the `rotateLookToward` position fix), anti-cheat doesn't false-positive. Then **tune the 1-v-4 matchup** (queue #2) вҖ” energy-fighter vs angles-fighter feel, turn radii, dive recovery вҖ” reasoning about the matchup on every number.

On every bird Model in `Workspace.Birds`, set *before parenting* by `GameServer.spawnBird`:
